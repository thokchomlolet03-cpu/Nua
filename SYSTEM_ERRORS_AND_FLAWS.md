## Active Issues (Newly Discovered)

The following issues were discovered during a deep technical analysis of the Node.js backend (`backend/src/`). They currently remain unresolved.

| ID | Severity | Description | File(s) |
|---|---|---|---|
| **B9** | 🔴 Critical | **FlatBuffers Schema Out-of-Sync**: The `schema/nua_schema.fbs` file was updated with the `directive:string` field, but the Node.js TypeScript schema definitions (`backend/src/schema/nua-serialization/time-segment.ts`) were not regenerated. `NuaBundler.ts` is calling `createTimeSegment` without the directive, meaning `.nuab` bundles generated by the backend will crash or corrupt on the Android client due to mismatched field indexing. | `backend/src/packager/NuaBundler.ts`, `backend/src/schema/` |
| **B10** | 🟡 Moderate | **Unhandled Rate Limits (HTTP 429)**: `TranslationAgent.ts` lacks retry logic or exponential backoff for Google GenAI calls. If the API hits a rate limit, it throws an unhandled error, crashing the ingestion job and failing the API response. | `TranslationAgent.ts` |
| **B11** | 🟡 Moderate | **Greedy Regex JSON Parsing**: `text.match(/\[[\s\S]*\]/)` is used to parse LLM JSON responses. This greedy regex can fail if the output contains trailing markdown noise, leading to catastrophic ingestion failures. | `TranslationAgent.ts` |
